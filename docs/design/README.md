# 设计文档模板

本目录用于存放长期有效的系统设计决策。设计文档应描述稳定边界、架构取舍、数据模型、接口合同、核心算法模型和跨模块约束，不记录短期执行清单。

## 建议文件命名

- `总体设计与技术选型.md`
- `数据模型与排考约束模型.md`
- `排考算法设计.md`
- `前后端接口设计.md`

## 模板

```markdown
# <设计主题>

## 1. 背景

说明该设计要解决的问题、输入文档和约束来源。

## 2. 目标与非目标

- 目标：
- 非目标：

## 3. 设计方案

描述核心方案、模块边界、数据流和关键接口。

## 4. 关键决策

记录已确认的技术选型、取舍理由和被拒绝的备选方案。

## 5. 风险与约束

列出实现风险、课程要求约束、部署约束和可测试性要求。

## 6. 后续影响

说明本设计会影响哪些计划、状态、测试和课程报告章节。
```
