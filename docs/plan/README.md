# 计划文档模板

本目录用于存放可执行计划。计划文档应描述阶段目标、任务拆分、验证命令和交付边界，不承载长期架构事实；长期事实应沉淀到 `docs/design/`，当前进展应同步到 `docs/status/`。

## 建议文件命名

- `排考算法原型计划.md`
- `开发环境与技能准备计划.md`
- `前端管理台实现计划.md`
- `课程报告整理计划.md`

## 模板

```markdown
# <计划主题>

## 1. 目标

用 1-3 句话说明本计划完成后应达到的可验证状态。

## 2. 范围

- 本轮包含：
- 本轮不包含：

## 3. 前置条件

列出需要先确认的文档、工具、数据、环境和风险。

## 4. 任务清单

- [ ] 任务 1：
- [ ] 任务 2：
- [ ] 任务 3：

## 5. 验证方式

列出最窄能证明本计划完成的命令、截图、报告材料或人工验收项。

## 6. 交付物

列出本计划应新增或修改的文件、演示结果和课程报告素材。
```
