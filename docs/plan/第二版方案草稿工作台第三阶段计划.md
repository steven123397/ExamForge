# 第二版方案草稿工作台第三阶段计划

## 1. 目标

在第二阶段已经完成草稿治理、发布确认、废弃和真实 PostgreSQL 验证的基础上，第三阶段聚焦方案工作台的实际调度效率：让运营人员可以通过拖拽完成单场考试调整，并由系统给出局部调整建议，最后补齐课程报告需要的流程图、状态图和演示脚本材料。

完成后，方案工作台应能演示“发现冲突 → 拖拽或采纳建议 → 即时校验 → 对比确认 → 发布”的更自然的人机协同流程。

## 2. 范围

- 本阶段包含：时间段 × 考场矩阵中的拖拽式单场考试调整。
- 本阶段包含：基于当前草稿的单场局部调整建议接口和前端建议面板。
- 本阶段包含：建议结果的硬约束校验摘要、评分预估和可一键应用能力。
- 本阶段包含：课程报告流程图、状态图和演示脚本材料。
- 本阶段包含：API 行为测试、前端类型检查和构建、调度器回归验证。
- 本阶段不包含：真实登录权限、异步队列、WebSocket、通知系统、导出文件、完整 CP-SAT 局部重排和数据库迁移治理。

## 3. 第二版剩余内容判断

按 `docs/design/第二版实现内容设计.md` 对照，第二版硬性交付已经基本完成：草稿模型、创建草稿、人工调整、硬约束校验、对比、废弃、发布确认、矩阵工作台、审计和发布查询均已落地。

仍未充分完成或可增强的内容如下：

- 拖拽式矩阵调整：设计文档允许先用点击选择和表单调整，但拖拽是明确的后续体验增强项。
- 局部重排和锁定考试：设计文档将其放在后续衔接中，适合作为第三阶段的核心增量。
- 课程报告图示材料：第二阶段已有文字素材，但还缺流程图、状态图和演示脚本这类可直接放入报告或答辩的材料。
- 专门仓储测试：第二阶段已有 API 行为测试和真实 PostgreSQL 验证，后续可视风险补 PostgreSQL 仓储级测试；本阶段暂不单独展开。

## 4. 任务清单

- [ ] 任务 1：补 API 行为测试，覆盖单场考试调整建议接口，包括无建议、存在建议、建议排序和建议应用后的草稿校验结果。
- [ ] 任务 2：实现局部调整建议服务，基于当前草稿、基础数据和硬约束校验生成候选时间段、考场和监考教师组合。
- [ ] 任务 3：新增建议 API，例如 `GET /api/schedule-drafts/:id/assignments/:examTaskId/suggestions`，返回候选安排、冲突数量、评分预估和解释。
- [ ] 任务 4：实现 Web 建议面板，在选中考试后展示候选调整，并支持一键应用建议。
- [ ] 任务 5：实现矩阵拖拽交互，将考试卡片拖到目标时间段和考场后复用既有 `PATCH` 草稿调整接口，并保留键盘和表单调整路径。
- [ ] 任务 6：补前端交互状态，覆盖拖拽悬停、不可放置、应用中、应用失败和硬冲突提示。
- [ ] 任务 7：补课程报告图示材料，包括 Mermaid 流程图、草稿状态图、演示脚本和测试结论摘要。
- [ ] 任务 8：同步 `docs/status/project_status.md`，记录第三阶段完成事实、验证结果和剩余风险。

## 5. 建议接口草案

```http
GET /api/schedule-drafts/:id/assignments/:examTaskId/suggestions
```

响应示例：

```json
{
  "draft": {
    "id": "draft-001",
    "status": "blocked"
  },
  "examTaskId": "e-database",
  "suggestions": [
    {
      "assignment": {
        "exam_task_id": "e-database",
        "room_id": "r-lab-2",
        "time_slot_id": "s-005",
        "teacher_ids": ["t-li"]
      },
      "hardConflictCount": 0,
      "score": 92,
      "reasons": ["满足实验机房要求", "监考教师无时间冲突"]
    }
  ]
}
```

建议生成先采用确定性枚举：遍历可用时间段和考场，保留原监考教师或枚举满足监考人数的教师组合，复用现有硬约束校验并按硬冲突数量、评分和资源利用情况排序。暂不引入新的 CP-SAT 局部求解器。

## 6. 验证方式

本计划完成前运行：

```bash
npm test
npm run typecheck
npm run build
cd apps/scheduler && uv run --python 3.12 --extra dev python -m pytest -q
git diff --check
```

前端交互还需要至少一次本地手动验证或 Playwright 截图验证，确认矩阵拖拽、建议应用和表单调整三条路径均能触发同一套草稿校验结果。
