# 第二版方案草稿工作台第一阶段计划

## 1. 目标

基于 `docs/design/第二版实现内容设计.md`，先完成一个可演示的排考方案草稿闭环：管理员可以从自动排考运行创建草稿，读取草稿详情，调整单场考试安排，查看即时硬约束冲突，在无硬冲突时发布草稿，并在 Web 方案工作台中用时间段 × 考场矩阵完成基础操作。

## 2. 范围

- 本轮包含：草稿领域类型、内存仓储、PostgreSQL 仓储、草稿 API、人工调整硬约束校验、审计事件、基础矩阵工作台和状态文档更新。
- 本轮包含：从 `schedule_runs` 复制生成草稿，草稿发布后复用已发布方案查询能力。
- 本轮不包含：复杂拖拽、局部重排、登录权限、消息通知、队列、WebSocket、生产级多租户和课程报告终稿。

## 3. 前置条件

- 已阅读根目录 `AGENTS.md`、`docs/AGENTS.md`、`apps/AGENTS.md`、`apps/scheduler/AGENTS.md`。
- 已阅读 `docs/index.md`、`docs/status/project_status.md` 和 `docs/design/第二版实现内容设计.md`。
- 当前 `main` 已同步到 `origin/main`，但工作区存在用户已有文档改动，需要保留并在其基础上继续。
- 调度器测试优先使用 `uv run --python 3.12 --extra dev python -m pytest -q`，因为本机默认 `python` 可能不满足 `apps/scheduler/pyproject.toml`。

## 4. 任务清单

- [x] 任务 1：先补 API 行为测试，覆盖创建草稿、读取详情、调整产生冲突、有冲突发布失败、修复后发布成功。
- [x] 任务 2：扩展共享领域类型，增加草稿摘要、详情、变更事件、对比和发布响应合同。
- [x] 任务 3：实现草稿仓储接口和内存仓储逻辑，包含确定性硬约束校验、状态迁移、审计事件和草稿发布。
- [x] 任务 4：实现草稿 API 路由和请求校验。
- [x] 任务 5：扩展 PostgreSQL schema、迁移和仓储实现，保证真实数据库路径具备同等草稿能力。
- [x] 任务 6：扩展 Web 运营台，新增方案工作台、草稿列表、矩阵视图、调整表单、冲突说明和发布确认状态。
- [x] 任务 7：同步 `docs/status/project_status.md`，记录第二版第一阶段的当前事实、验证结果和剩余风险。

## 5. 验证方式

本计划完成前运行：

```bash
npm test
npm run typecheck
npm run build
cd apps/scheduler && uv run --python 3.12 --extra dev python -m pytest -q
git diff --check
```

如环境阻塞，需要记录具体命令、退出原因和替代验证结果。
