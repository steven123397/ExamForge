# ExamForge 第二版部署与演示说明

## 1. 本地环境

推荐环境：

- Node.js 20 或更高版本。
- npm workspace。
- Python 3.12 或更高版本。
- `uv` 用于调度器测试环境。
- 可选 PostgreSQL 16；未配置 `DATABASE_URL` 时 API 使用内置演示仓储。

## 2. 安装依赖

```bash
npm install
```

调度器测试使用：

```bash
cd apps/scheduler
uv run --python 3.12 --extra dev python -m pytest -q
```

## 3. 启动开发环境

仓库根目录运行：

```bash
npm run dev
```

默认地址：

- Web：`http://127.0.0.1:3000`
- API：`http://127.0.0.1:4000`
- 健康检查：`http://127.0.0.1:4000/health`

## 4. PostgreSQL 演示路径

如需使用 PostgreSQL：

```bash
docker compose up -d postgres
npm run seed --workspace @examforge/db
DATABASE_URL=postgres://examforge:examforge@127.0.0.1:5432/examforge npm run dev --workspace @examforge/api
```

迁移文件位于 `packages/db/drizzle/`。当前课程演示可直接使用 seed 数据；重复执行旧迁移时若对象已存在，应以数据库当前 schema 为准。

## 5. 验证命令

第四阶段收口验证矩阵：

```bash
npm test
npm run typecheck
npm run build
npm run test:e2e
cd apps/scheduler && uv run --python 3.12 --extra dev python -m pytest -q
git diff --check
```

## 6. 课堂演示脚本

1. 打开 Web 运营台，展示批次概览、考试任务、教师、考场、时间段数量。
2. 点击“运行排考”，展示同步排考结果、冲突解释和资源利用率。
3. 点击“异步排考”，展示后台作业从 queued/running 到 completed，并生成新的运行版本。
4. 在运行历史中创建草稿，进入“方案工作台”。
5. 在矩阵中选择考试，查看局部调整建议，点击“应用”。
6. 锁定当前考试，说明锁定后禁止人工调整；再解锁并展示可继续调整。
7. 制造或加载有冲突草稿，点击“局部再平衡”，展示未锁定冲突考试被自动修复。
8. 查看发布确认面板，确认硬冲突为 0 后发布草稿。
9. 在“已发布查询”中按教师或学生群体查询安排。
10. 刷新通知预览并导出 CSV，说明发布后对外分发路径。

## 7. 角色演示

Web 顶部角色选择用于演示轻量权限护栏：

- 管理员：可以发布、回滚、删除和维护数据。
- 排考员：可以排考、创建草稿、调整草稿和维护教师不可用时间。
- 只读：不能执行排考和变更类操作。

当前角色通过 `x-examforge-role` 请求头传递，后续可替换为真实登录与权限系统。
