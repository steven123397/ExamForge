**一、课程周期与提交**

**（一）设计周期**

小学期内单人独立完成，禁止组队分工、抄袭文档图表，所有分析、建模、设计内容独立完成。

于8月5日之前，提交至邮箱：44033102@qq.com

**（二）提交成果**

提交完整软件工程课程设计大报告Word文档，无需提交源代码、数据库脚本、可运行程序。

提交压缩包命名：学号_姓名_选题名称。

压缩包内仅包含：课程设计报告Word文档（含全部文字、图表、附录测试用例）。

**二、课程设计总体要求**

报告须覆盖软件完整生命周期，章节顺序固定，全程侧重软件分析与设计，不要求编程实现，报告提纲：

**1. 引言（项目背景、开发目标、目标用户、预设开发环境与技术栈）**

**2. 可行性分析（技术可行性、经济可行性、操作可行性、法律可行性）**

**3. 需求分析**

**3.1 功能性需求**

要求给出：

（1）核心业务流程文字描述；

（2）面向对象建模（参与者、完整用例图、每条用例配套用例规约）。

**3.2 非功能性需求**

要求给出：

性能、易用性、安全性、可维护性、兼容性等非功能性需求描述。

**4. 系统总体设计**

要求给出：

（1）系统分层架构图、整体功能模块拆分结构图；

（2）数据库 E-R 图、完整数据表结构设计（字段名、类型、主键、外键、约束说明）。

**5. 详细设计**

要求给出：

（1）核心业务类图，必须体现封装、继承、多态等面向对象特征

（2）核心业务流程时序图 / 活动图

（3）模块内部逻辑、模块间接口设计说明

**6. 测试方案设计**

要求给出：

（1）测试分类：单元测试、集成测试、系统测试规划

（2）大批量标准化测试用例，覆盖正常场景、边界场景、异常错误场景

（3）缺陷分类、缺陷记录与修复预案设计

**7. 软件维护规划**

要求给出：

纠错性维护、适应性维护、完善性维护、预防性维护实施方案

**8. 总结与展望**

要求给出：

项目设计难点、课程收获、系统可优化拓展方向

**三、格式规范要求**

**1. UML 图表绘制规范，统一编号、图例标注清晰；**

**2. 数据库至少设计 3 张及以上数据表，包含关联关系；**

**3. 系统划分至少两类用户角色，完成权限隔离设计；**

**4. 报告排版统一：宋体小四、1.5 倍行距、图表编号、参考文献格式规范。**

**四、各类型题目设计要求**

**（一）校园管理系统设计要求**

1. 双角色权限划分：管理员、普通师生用户，完整权限区分设计；

2. 完整基础数据增删改查业务需求，支持多条件模糊检索；

3. 内置自动计算业务规则（绩点计算、逾期罚款、预约超时释放等）；

4. 数据统计、报表查询功能需求设计；

5. 数据备份、基础数据重置安全方案设计。

**（二）小型商业业务管理系统设计要求**

1. 操作员分级权限体系；

2. 业务单据留存、历史记录追溯流程设计；

3. 库存/资金统计、阈值预警需求设计；

4. 周期业务汇总统计功能；

5. 异常操作拦截规则（负库存、重复订单、非法金额等）。

**（三）带算法功能系统设计要求**

除基础管理模块外，完成：

1. 核心算法原理文字说明；

2. 算法逻辑流程图独立设计；

3. 算法专项测试用例、效果对比分析；

4. 算法优化改进思路设计。

**（四）轻量化桌面工具设计要求**

1. 数据分类、多条件筛选需求；

2. 消息提醒、状态标记功能设计；

3. 本地数据存储、导入导出方案；

4. 基础数据统计展示模块。
